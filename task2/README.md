## 1. Диаграмма активности: Оформление заказа

### Описание процесса
Диаграмма активности описывает процесс выполнения заказа от момента выбора блюд до получения доставки.

1. **Customer** начинает с добавления блюд в корзину (`addToCart`), после чего система проверяет доступность ингредиентов.
2. Пользователь выбирает способ оплаты и доставки (`selectPaymentMethod`), система рассчитывает итоговую сумму.
3. **RestaurantManager** получает уведомление и может подтвердить или отклонить заказ (`confirmOrder`).
4. Если заказ подтверждён — начинается приготовление (`startCooking`), затем передаётся курьеру.
5. **Courier** принимает заказ и доставляет клиенту (`deliverOrder`).
6. Если заказ отклонён или таймаут — пользователю предлагается отмена или повторный заказ.
7. После доставки пользователь может оставить отзыв (`addReview`).

## 3. Диаграмма последовательности (Sequence)

Диаграмма последовательности детализирует временной порядок взаимодействий между актёром Customer и объектами Order, PaymentService, RestaurantSystem, CourierService.
После нажатия кнопки «Заказать» активируется объект Customer, который создаёт Order.
Order вызывает processPayment() у PaymentService.
PaymentService отправляет запрос в платёжный шлюз и возвращает статус.
RestaurantSystem получает уведомление и подтверждает возможность приготовления.
Альтернативный фрагмент (alt) разделяет сценарии:
При успехе: создаётся задание для CourierService, пользователь получает трек-номер.
При ошибке: отмена заказа, возврат средств, уведомление пользователя.
Фрагмент opt (опционально): если доставка задерживается — начисление бонусов.

## 4. Диаграмма состояний: Заказ (Order)
### Описание жизненного цикла

Диаграмма состояний описывает жизненный цикл заказа в системе доставки еды (**Order**).

1. **Начальное состояние `Created`** достигается после вызова `placeOrder()`.
2. Из **`Created`** переход в **`PaymentPending`** осуществляется методом `initiatePayment()` (ожидание подтверждения оплаты).
3. Внутри **`PaymentPending`** возможны переходы:
   - При успешной оплате → **`Confirmed`** (метод `confirmPayment()`)
   - При ошибке оплаты → **`Cancelled`** (метод `cancelOrder()`)
4. Из **`Confirmed`** переход в **`Cooking`** после подтверждения ресторана (`restaurantConfirm()`).
5. Внутри **`Cooking`** заказ готовится (время ~20-40 мин).
6. При готовности → **`ReadyForPickup`** (метод `markReady()`).
7. Курьер забирает заказ → **`OnDelivery`** (метод `courierPickup()`).
8. При доставке клиенту → **`Delivered`** (метод `confirmDelivery()`).
9. Из **`Delivered`** возможен переход в **`Completed`** после оставления отзыва (`addReview()`).
10. Из любого состояния возможен переход в **`Cancelled`** (пользователь/ресторан/система).
